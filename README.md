# Сравнение поз и движений по видео — Виртуальный тренер

## Описание проекта

Цель проекта — разработка системы, которая позволяет сравнивать движения человека на видео с эталонной моделью. Такой подход может быть использован для оценки правильности выполнения упражнений, обучения движениям и создания "виртуального тренера".

---

## Основные цели и задачи

- Определить ключевые точки скелета человека на видео с помощью модели `Keypoint R-CNN`.
- Сравнить позы покадрово между двумя видеофрагментами.
- Вычислить метрики различий: косинусное сходство и взвешенное расстояние.
- Визуализировать ошибку повторения движения.
- Интерпретировать результат и дать оценку точности исполнения.

---

## Данные

- **Формат:** Видео `.mp4`, содержащие эталонную запись и движения пользователя.
- **Обработка:** Кадры извлекаются с шагом (по умолчанию `FRAME_STEP=10`).

---

## Этапы реализации проекта

1. Оценка поз на изображениях при помощи модели `KeypointRCNN` из библиотеки `torchvision` ([images.ipynb](myproject_test_and_utils/images.ipynb)).
2. Оценка поз на предложенных видеофрагментах ([video.ipynb](myproject_test_and_utils/video.ipynb)).
3. Выбор сферы применения (обучение технике бокса), подбор обучающего видео, подготовка пользовательского видео.
4. Создание отдельных .py файлов для загрузки модели, загрузки видео, применения модели и вывода результирующего видео, содержащего текущие значения метрик на каждом кадре и дополнительный кадр с общей оценкой.
5. Создание функции для сохранения графиков покадровых метрик косинусного сходства и средневзвешенного расстояния.

---

## Запуск проекта

Запуск проекта осуществляется при помощи запуска скрипта [`main.py`](scripts/main.py). Список возможных аргументов:

- `--ref` - путь к референсному видео (тренер);
- `--user` - путь к видео пользователя;
- `--output` - путь для сохранения выходного видео;
- `--step` - шаг обработки кадров (по умолчанию - `FRAME_STEP` из [config.py](scripts/config.py));
- `--conf` - порог уверенности детекции (0..1) (по умолчанию - `CONF_THRESHOLD` из [config.py](scripts/config.py));
- `--skelet` - накладывать скелет тренера (зелёный) на видео пользователя. Если не указан, выводятся только метрики;
- `--progress` - показывать прогресс обработки видео моделью;
- `--ru` - использовать русский язык для текста на последнем кадре (были проблемы с кодировкой);
- `--plot` - сохранить графики метрик в указанный файл.

Пример:

```bash
# В корне проекта
python scripts/main.py --ref data/ref.mp4 --user data/user.mp4 --output output/result --skelet --ru --progress --plot output/metrics.png
```

## Вывод результата

В результате работы основного скрипта создается выходной видеофайл, который содержит видео пользователя с наложенными метриками косинусного сходства и взвешенного расстояния и дополнительным кадром с выводом средних значений метрик и итоговой оценкой "Отлично", "Хорошо", "Средне" или "Плохо". Опционально можно включить наложение скелета тренера на видео пользователя, а также сохранить графики метрик в файл.

Также был создан прототип мобильного приложения под ОС Android (но модель Keypoint R-CNN слишком тяжелая для использования на смартфоне, в идеале её заменить на более легкую).
